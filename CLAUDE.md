## Overview

**jekyll-theme-purple** is a Jekyll theme — a purple-styled fork of [Chirpy](https://github.com/cotes2020/jekyll-theme-chirpy). It is distributed as a Ruby gem and has a dual build pipeline: Ruby/Jekyll for the static site, and Node.js for compiling JavaScript (Rollup + Babel) and purging CSS (PurgeCSS).

## Setup

```bash
bundle install
npm install
npm run build   # must run before serving — compiles JS and purges CSS
```

## Development Commands

| Task | Command |
|---|---|
| Serve site (dev, live reload) | `bash tools/run.sh` |
| Serve in production mode | `bash tools/run.sh --production` |
| Watch JS changes | `npm run watch:js` |
| Watch CSS changes | `npm run watch:css` |
| Watch both (parallel) | `npm run watch` |
| Full Docker dev stack | `docker compose up` |
| Lint JS | `npm run lint:js` |
| Lint SCSS | `npm run lint:scss` |
| Auto-fix SCSS lint | `npm run lint:fix:scss` |
| Run all linters | `npm test` |
| Build + HTML proof test | `bash tools/test.sh` |

`bash tools/run.sh` binds to `127.0.0.1:4000` by default. Use `-H 0.0.0.0` to expose externally.

`npm run build` must be re-run whenever `_sass/**/*.scss` or `_javascript/**/*.js` changes, unless using `npm run watch`.

## Architecture

### Dual Build Pipeline

The theme separates concerns into two pipelines that must both produce output before the site renders correctly:

1. **JavaScript** — source in `_javascript/`, bundled by Rollup into `assets/js/dist/*.min.js`. Entry points are `commons`, `home`, `categories`, `page`, `post`, `misc`, `theme`. PWA files (`pwa/app.js`, `pwa/sw.js`) have Jekyll YAML frontmatter injected at build time so Jekyll can serve them with the correct permalink. `_javascript/modules/` holds shared logic split into `components/` and `layouts/`.

2. **CSS** — `purgecss.js` strips unused Bootstrap classes by scanning `_includes/**/*.html`, `_layouts/**/*.html`, and `_javascript/**/*.js`, then writes the purged result to `_sass/vendors/_bootstrap.scss`. The main entry is `assets/css/jekyll-theme-purple.scss`, which imports either `main.scss` (development) or `main.bundle.scss` (production) based on `jekyll.environment`.

### SCSS Structure

```
_sass/
  abstracts/   # variables, mixins, placeholders, breakpoints
  themes/      # _light.scss and _dark.scss — color scheme definitions
  base/        # resets, typography, etc.
  components/  # buttons, popups
  layout/      # sidebar, topbar, footer, panel
  pages/       # home, post, categories, tags, etc.
  vendors/     # auto-generated by purgecss.js (gitignored in dev)
```

Purple's visual identity lives in `_sass/themes/`. `_sass/abstracts/_variables.scss` defines layout dimensions and font families.

### Asset Origins

Third-party libraries (tocbot, glightbox, MathJax, etc.) are resolved from one of two sources controlled by `assets.self_host.enabled` in `_config.yml`:
- `_data/origin/cors.yml` — CDN URLs (used in production)
- `_data/origin/basic.yml` — self-hosted paths under `/assets/lib/` (used in development)

### Liquid Templates

`_layouts/` provides page-type templates (`default`, `home`, `post`, `page`, `archives`, `category`, `tag`, `tags`). `_includes/` provides reusable partials. The `js-selector.html` include maps page layout names to the correct JS bundle, and `origin-type.html` switches between CDN and self-hosted asset paths.

### Data Files

- `_data/locales/` — UI strings for each supported language
- `_data/authors.yml` — author definitions for multi-author blogs
- `_data/share.yml` — social share button configuration
- `_data/media.yml` — media embedding configuration
- `_data/contact.yml` — sidebar contact icons

## Commit & Release Conventions

Commits must follow **Conventional Commits** (enforced by commitlint via Husky):

```
feat: add new feature
fix: fix a bug
perf: performance improvement
refactor: code change (hidden in changelog)
chore: maintenance tasks
```

Releases are automated via `semantic-release` and triggered only on the `production` branch. The changelog is written to `docs/CHANGELOG.md`. Do not push directly to `production`.

## Gem Distribution

`jekyll-theme-purple.gemspec` controls which files are included in the gem (only `_includes/`, `_layouts/`, `_sass/`, `_data/locales/`, `_data/origin/`, and `assets/`). When `assets/js/dist/` and `_sass/vendors/` are built and committed (done by `tools/init.sh`), they are included in the gem.
